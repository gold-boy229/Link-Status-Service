## Запус сервиса
1. Клонировать проект:
`git clone https://github.com/gold-boy229/Link-Status-Service .`
2. Создать .env файл с переменными, указанными в .env_example одним из способов:
    1.  На Windows: `copy .env_example .env`
    2.  На Linux: `cp .env_example .env`
    3. Свой вариант
3. Опицонально. Задайте нужные вам значения переменным в .env файле
4. Выполните команду `go run ./cmd/app/main.go` из корня проекта

## Изначальные технические требования 
ТЗ можно найти в ./docs

## Описание проекта
Описание будет представлено в алфавитном порядке названия папок в корне проекта и за что каждая из этих папок отвечает
1.  .github - содержит два CI pipeline'а, для автозапуска тестов и линтера
1.  .vscode - содержит локальные настройки IDE, в частности настройки линтера и форматтера Go кода
1.  cmd - содержит точку входа в приложение
1.  data - вспомогательная директория (изначально отсутствует), в которую сохраняются данные сервиса при его выключении
1.  docs - содержит изначальное ТЗ
1.  internal - рассмотрим ниже 
1. Остальное - файлы, лежащие в корне проекта, названия которых говорит за себя.

### Структура папки /internal
Описание будет представлено в алфавитном порядке названия папок внутри /internal:
1.  app - содержит объявление приложения и всех его зависимостей, задание эндпоинтов, реализацию graceful shutdown'a, список интерфейсов, используемых в этом пакете
1.  client - содержит обертку поверх http.Client'a для которой определен метод, проверяющий доступность интернет-ресурса по ссылке
1.  config - содержит работу .env переменными
1.  consts - содержит глобальные константы проекта
1.  dto - содержит DTO структуры, которые используются для получения/отправки данных внутри HTTP-обработчиков
1.  entity - содержит структуры бизнес-логики
1.  handlers - содержит HTTP-обработчики
1.  mocks - содержит mock'и зависимостей для unit тестов
1.  pdf - содержит всю реализацию постороения .pdf файла
1.  repository - содержит реализацию репозитория
1.  service - содержит всю бизнес-логику. 
1.  utils - содержит утилиты/функции, которые используются в разных пакетах
